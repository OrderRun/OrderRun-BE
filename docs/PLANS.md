# 계획 가이드

## 목적

실행 계획은 넓은 요청을 경계가 분명하고 검증 가능한 작업 단위로 바꾸기 위한 문서다.

## 위치 규칙

- 진행 중 계획: `docs/exec-plans/active/`
- 완료된 계획: `docs/exec-plans/completed/`
- 장기 부채나 보류 항목: `docs/exec-plans/tech-debt-tracker.md`

## 권장 템플릿

각 계획 문서는 아래 내용을 포함한다.

1. 목표
2. 범위와 비범위
3. 가정
4. 작업 분해
5. 검증 전략
6. 롤아웃 또는 롤백 메모
7. 남은 리스크

## 파일명 규칙

계획 문서는 `YYYY-MM-DD-short-title.md` 형식을 사용해 정렬과 추적이 쉽도록 한다.
