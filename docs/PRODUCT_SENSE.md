# 제품 감각 가이드

## 제품 가설

`OrderRun`은 일을 맡기려는 사용자와 빠르고 안전하게 수행할 수 있는 사용자를 연결하는 과정의 마찰을 줄여야 한다.

## 우선순위 기준

- 편의보다 신뢰를 먼저 본다. 인증, 권한, 상태 무결성이 우선이다.
- 예외보다 핵심 루프를 먼저 완성한다. 요청 생성, 제안, 수락, 수행, 정산이 우선이다.
- 인수인계 지점의 모호성을 줄인다. 요청 내용, 가격, 상태, 책임 주체를 명확히 한다.
- 초기 단계에서는 기능 수보다 흐름의 명확성을 우선한다.

## 결정 전 질문

새 작업을 추가하기 전에 아래를 확인한다.

1. 이 작업이 핵심 매칭 루프를 강화하는가
2. 운영자나 사용자 혼란을 줄이는가
3. 하나의 제품 스펙으로 명확히 설명할 수 있는가
4. 같은 학습 가치를 더 단순한 방식으로 얻을 수 없는가
