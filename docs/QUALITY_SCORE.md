# 품질 점수 가이드

## 목적

기능이나 변경이 머지 또는 데모 가능한 상태인지 판단할 때 사용하는 기준표다.

## 평가표

| 영역 | 질문 | 목표 |
| --- | --- | --- |
| 정확성 | 문서화된 흐름과 실제 동작이 일치하는가 | 알려진 불일치 없음 |
| 검증 가능성 | 검증 경로가 명확한가 | 반복 가능하고 가능하면 자동화 |
| 운영 가능성 | 실패를 관찰하고 원인을 추적할 수 있는가 | 로그와 에러가 행동 가능 |
| 일관성 | 기존 도메인과 API 패턴에 맞는가 | 놀라운 규칙 없음 |
| 보안 | 인증, 데이터 접근, 비밀 정보 처리가 안전한가 | 명백한 공백 없음 |
| 단순성 | 더 뻔한 대안보다 구조가 작은가 | 불필요한 이동 부품 최소화 |

## 점수 기준

- `5`: 바로 사용 가능하고 뒷받침 근거가 충분함
- `3`: 사용 가능하지만 후속 부채가 명시되어 있음
- `1`: 리스크가 크거나, 불명확하거나, 검증이 약함

어느 한 항목이라도 `1`이면 완료로 간주하지 않는다.
