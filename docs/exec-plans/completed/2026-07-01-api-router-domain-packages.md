# API 라우터 도메인 패키지화

## 목표

`app/api/v1`의 평면 모듈을 도메인별로 분류하고, 모듈 이름에서 라우터
책임이 드러나도록 한다.

## 반영 내용

- 기존 라우터를 문서상 도메인 경계에 맞는 패키지로 이동했다.
- 모든 라우터 모듈을 `*_router.py`로 명명했다.
- `app/api/v1/router.py`가 `/v1` prefix와 하위 라우터 조립을 담당한다.
- `app/main.py`는 조립된 `api_router` 하나만 등록한다.
- 내부 import와 monkeypatch 경로를 새 구조에 맞게 변경했다.

## 검증 기준

- 기존 공개 API 경로와 OpenAPI 계약이 유지된다.
- 저장소에 이전 평면 API 모듈 import가 남지 않는다.
- 전체 통합 테스트가 테스트용 MySQL 환경에서 통과한다.

## 남은 리스크

관리자 API는 여러 업무 도메인을 포함하지만 이번 변경에서는 기존
액터 경계를 유지했다. 규모가 커지면 별도 설계 작업으로 분리를 검토한다.
