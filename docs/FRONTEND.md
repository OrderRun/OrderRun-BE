# 프론트엔드 연동 노트

## 범위

이 저장소는 백엔드 중심이지만, 프론트엔드와 맞닿는 계약면은 명시적으로 관리해야 한다.

## 원칙

- OpenAPI와 제품 스펙을 프론트엔드 연동의 기준 문서로 본다.
- 응답 envelope과 에러 코드 의미는 가능한 한 안정적으로 유지한다.
- 초기 단계에서는 파괴적 변경보다 확장 가능한 추가 변경을 우선한다.
- 임시 필드나 임시 플로우라도 외부에 노출되면 문서에 먼저 적는다.

## 주요 연동 지점

- 인증과 세션 시작
- 요청서 목록과 상세 조회
- 제안 등록과 수락 흐름
- 미션 진행과 완료 상태
- 결제와 정산 상태 표시

## 핸드오프 규칙

백엔드 변경이 UX 흐름에 영향을 주면 아래 두 문서를 함께 갱신한다.

- `docs/product-specs/`: 사용자 기대 행동
