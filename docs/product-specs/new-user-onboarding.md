# 신규 사용자 온보딩

## 목표

신규 사용자는 서비스에 진입한 뒤 양면 마켓 구조를 이해하고, 계정을 만들고, 혼란 없이 다음 행동이 가능한 초기 상태에 도달해야 한다.

## 사용자 유형

- 요청을 만들고 싶은 고객
- 나중에 일을 수락하고 싶은 수행자

## 성공 기준

- 사용자가 계정을 만들거나 유효한 세션을 획득한다.
- 고객과 수행자 행동의 차이를 이해한다.
- 첫 세션 안에 의미 있는 다음 단계로 진입한다.

## 핵심 흐름

1. 사용자는 서비스 랜딩에서 마켓플레이스의 핵심 가치를 짧게 이해한다.
2. 사용자는 회원가입 또는 로그인한다.
3. 사용자는 최소 필수 프로필 정보를 입력한다.
4. 사용자는 최신 필수 약관에 동의한다.
5. 사용자는 가장 관련성 높은 첫 행동으로 이동한다.

고객의 첫 행동:
요청서 생성

수행자의 첫 행동:
기회 탐색 또는 프로필 보완

## 제품 규칙

- 온보딩은 다음 행동을 여는 데 필요한 정보만 요구한다.
- 사용자가 이후 두 역할을 모두 수행할 수 있어도 역할 설명은 명시적으로 제공한다.
- 약관 동의 버전은 추적 가능해야 한다.
- 미완료 온보딩 상태는 이어서 재개 가능해야 한다.

## 열린 질문

- 회원가입 시점에 고객/수행자 의도를 명시적으로 물을지
- 첫 거래 전에 휴대폰 인증을 강제할지
- 프로필 완성도가 제안 등록을 막아야 하는지
