# 신뢰성 가이드

## 목표

백엔드는 핵심 생명주기 전반에서 상태 무결성을 지켜야 한다.

- 요청서 공개
- 제안 생성과 수락
- 미션 진행
- 결제 상태 변경

## 기본 기준

- 상태 전이는 서버에서 검증해야 한다.
- 쓰기 작업은 트랜잭션으로 보호되어야 한다.
- 외부 실패는 예측 가능한 에러 코드로 드러나야 한다.
- 로그에는 실패한 리소스와 전이 정보가 식별 가능하게 남아야 한다.

## 주요 실패 모드

- 중복 제안 수락
- 미션 상태와 결제 상태의 불일치
- 재시도 요청에서의 멱등성 공백
- 상태 변경 경로의 권한 검증 누락

## 검증 기대치

- 상태 전이에 대한 단위 또는 서비스 테스트
- 권한 경계에 대한 API 수준 검증
- 과거 실패 흐름에 대한 명시적 회귀 검증
